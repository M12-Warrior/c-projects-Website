# Where and How You’ll Edit Once This Is the Live Site

Short answer: **most of your day-to-day editing happens in your browser**, in the **Admin Dashboard** — similar to WordPress, but built into this site. You do **not** need to edit code or talk to me for normal updates.

---

## What you do in the browser (no code, no Cursor)

After the site is live at mile12warrior.com, you log in with your **admin account** and go to:

**https://mile12warrior.com/admin**

There you’ll use the **tabs**:

| Tab        | What you do |
|-----------|----------------------------------------------------------------|
| **Dashboard** | See counts: users, posts, orders, revenue, etc. |
| **Blog**      | **Add new posts**: click “New Post”, fill in Title, Excerpt, Content, check “Published” if you want it live, then “Create Post”. You can **delete** posts from the table. (Editing an existing post is not in the UI yet; we can add an “Edit” button if you want.) |
| **Messages**  | See **contact form submissions**: name, email, subject, message. You can mark as read and delete. To **respond**, use your own email app and reply to the email address shown in the message. |
| **Shop**      | Add new products, see orders, update order status. |
| **Forum**     | See threads, pin/lock/delete. |
| **Users**     | See users, change roles (user/admin), delete. |

So:

- **New blogs** → Admin → Blog → New Post.  
- **Contact / “emails”** → Admin → Messages (read there; reply from your email).  
- **Data** (orders, users, messages) → All visible and manageable in Admin.

You do **not** need to open this project in Cursor or edit any files for that.

---

## When you *do* need to work with me (or a developer)

Use Cursor (and me) when you want to:

- Change the **look** of the site (layout, colors, new sections).
- Add a **new kind of page** or feature (e.g. a new form, a new section on the homepage).
- Change **how something works** (e.g. how the contact form sends mail, or where messages go).
- Fix **bugs** or odd behavior.
- Add an **“Edit post”** flow in the admin so you can edit existing blog posts in the browser.

Those are code/project changes. Day-to-day content and messages are handled in the Admin Dashboard.

---

## Comparison to WordPress

| In WordPress now           | In this site once live        |
|----------------------------|--------------------------------|
| Log in to wp-admin         | Log in at mile12warrior.com → go to /admin |
| Write posts in the editor  | Admin → Blog → New Post        |
| Comments / contact         | Contact form → Admin → Messages; you reply from your email |
| Plugins, themes            | No plugins; changes = ask for code updates |

You’re not losing the ability to edit — you’re switching from WordPress’s dashboard to **this site’s Admin Dashboard** for content and data. No extra “thing to create” is required for you to do that; the admin is already part of what we built.

---

## Summary

- **Edits and updates (blogs, messages, data)** → In the browser at **mile12warrior.com/admin** after you’re live.  
- **Responding to “emails”** → Open Admin → Messages, then reply to the person’s email from your own email.  
- **Bigger changes (design, new features)** → We do those here in the project (you and me in Cursor).

If you want, we can add an **Edit** button for blog posts in the admin so you can change existing posts without touching code.
